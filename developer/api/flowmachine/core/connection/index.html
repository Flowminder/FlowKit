flowmachine.core.connection¶

Source: flowmachine/core/connection.py

Class(es) representing our connection to the database, along with utilities regarding the database.

Class Connection¶

Connection(port=None, user=None, password=None, host=None, database=None, pool_size=5, overflow=10, *, conn_str=None)
Source: flowmachine/core/connection.py

Establishes a connection with the database and provide methods for fetching data.

Attributes¶

  • available_tables

  • location_table

  • location_tables

  • subscriber_tables

Parameters¶

  • port, user, password, host, database: str

    Connection info for the database, as used by sqlalchemy.

  • pool_size: optional, int

    Number of connections to the db to use

  • overflow: optional, int

    Number of connections to the db to open temporarily

Note

sqlalchemy will open at most pool_size+overflow connections to the db and requests for more will time out rapidly. sqlalchemy will not immediately open pool_size connections, but will always keep that many open once they have been. You will, ordinarily be OK to ignore this setting.

Parameters¶

  • start: typing.Union[NoneType, str], default None

    If specified, list only available dates after this one (inclusive) as iso format date string

  • stop: typing.Union[NoneType, str], default None

    If specified, list only available dates before this one (inclusive) as iso format date string

  • table: typing.Union[str, typing.Tuple[str]], default calls

    Name(s) of tables to check, checks all tables listed under self.subscriber_tables if 'all' is passed.

  • strictness: int, default 0

    Three levels of strictness are available - 0 checks only that the table exists. 1 checks the approximate number of rows in the table is non-zero, and 2 checks performs a select on the actual table.

  • schema: str, default events

    Schema the table belongs to, defaults to 'events'

Returns¶

    Dict with tables as keys, containing lists of dates which are present

has_date¶

has_date(self, date, table, strictness=2, schema="events")
Source: flowmachine/core/connection.py

Check against the database tables with varying strictness whether there is data present for the provided date.

Parameters¶

  • date: datetime

    Date to check for

  • table: str

    Table under the event schema

  • strictness: {0, 1, 2}

    Three levels of strictness are available - 0 checks only that the table exists. 1 checks the approximate number of rows in the table is non-zero, and 2 tries to retrieve some rows from the table.

  • schema: str

    Schema the table belongs to

Returns¶

  • bool

    True if the date is available based on strictness criteria
